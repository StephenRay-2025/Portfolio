ADR-003: Why Machine Learning Is Deferred
Status: Accepted
Context
The project’s primary goal is to provide a reliable upgrade‑decision analyzer for AM4 players. During the technical spike we explored the available datasets and identified several potential ML use cases (e.g., predicting route profitability, clustering aircraft by performance). However, introducing ML would add significant complexity in terms of dependencies, model training, validation, and explainability.
Decision
We decided to defer machine learning to a future version (v0.2.0 or later). The MVP focuses on deterministic, transparent calculations based on the official dataset metrics (profit per flight, fuel consumption, CO₂ emissions, etc.). This keeps the tool easy to understand, audit, and maintain.
Consequences
- Positive: Minimal dependencies (pandas, streamlit, Parquet engine).
- Positive: Clear, reproducible logic that users can verify manually.
- Positive: Faster development and easier deployment.
- Negative: Misses opportunities for advanced insights (e.g., route demand forecasting). These can be added later as optional extensions.
Note: The DatasetService abstraction makes it easy to integrate ML models later without changing the data access layer.
